# Converged Decision Packet

Decision: Seed this repo as a docs-first teaching starter.

Rationale: A runtime-heavy implementation would make the first clone harder to understand. The useful primitive is the durable handoff artifact, not any specific private repo automation.

Decision: Include one completed Program and one completed child ExecPlan.

Rationale: Learners need to see what "done" looks like, including frontmatter, progress, validation, and retrospective.

Decision: Use a no-dependency Bun validator.

Rationale: Students should be able to run proof immediately after cloning or copying the folders.
