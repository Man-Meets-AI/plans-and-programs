# Dependency Graph

1. Root orientation files establish the repo purpose.
2. Program and ExecPlan contracts define the artifact schema.
3. Templates make the contracts easy to copy.
4. Prompts turn the contracts into repeatable agent workflows.
5. Skills expose those workflows to Codex-style agents.
6. The completed seed Program and ExecPlan demonstrate the whole loop.
7. Validation checks the required files and core schema.
